   12.1 В системе все доходы предприятия можно разделить на:
      1. доходы от основной деятельности предприятия и доходы от других видов деятель- ости
      2. доходы предприятия от продаж и дополнительные доходы предприятия
      3. бизнес-доходы и дополнительные доходы


   12.2 Учет выручки выполняется:
      1. вручную
      2. автоматически при регистрации хозяйственных операций продажи товара
      3. возможны оба варианта


   12.3 Выручкой признается:
      1. стоимость проданных товаров в ценах номенклатуры
      2. сумма фактически полученных денежных средств от клиентов за проданные товары
      3. возможны оба варианта. Это зависит от настроек параметров учета


   12.4 Выберите НЕверное утверждение:
      1. В программе учет выручки ведется при продаже товаров оптом, в розницу, при ко- иссионной торговле
      2. Вся выручка, независимо от валюты продажи, пересчитывается в валюту управлен- еского учета
      3. Выручка учитывается в валюте управленческого учета с НДС и без НДС, а также егламентированного учета без НДС
      4. Выручка может также учитываться в валюте взаиморасчетов с партнерами, для это- о нужно установить соответствующую функциональную опцию


   12.5 Прочие доходы могут возникать при отражении таких операций, как:
      1. поступление денежных средств (наличных или безналичных)
      2. списание задолженности (в части кредиторской задолженности предприятия)
      3. оприходование товаров
      4. Верны утверждения 2 и 3
      5. Верны все утверждения


   12.6 В какой валюте учитываются прочие доходы?
      1. в валюте взаиморасчетов
      2. в валюте управленческого учета с НДС
      3. в валюте управленческого учета без НДС
      4. в валюте регламентированного учета без НДС
      5. Верны утверждения 2, 3 и 4
      6. Верны все утверждения


   12.7 Выберите НЕверное утверждение:
      1. Статьи доходов используются для единой классификации всех отражаемых в при- ладном решении доходов
      2. Статья доходов – произвольный признак, по которому можно сгруппировать прочие оходы предприятия
      3. Перечень статей прочих доходов обычно утверждается руководством предприятия
      4. Доходы, относимые к одной статье, будут распределяться на прибыль указанных одразделений по общему правилу


   12.8 Дополнительной аналитикой статьи дохода может являться:
      1. организация
      2. подразделения
      3. направления деятельности
      4. договоры кредитов (депозитов)
      5. Верны утверждения 1 и 3
      6. Верны утверждения 1, 2, 3 и 4


   12.9 В конце месяца все прочие доходы:
      1. подлежат распределению по направлениям деятельности
      2. подлежат отнесению на организацию
      3. распределяются по подразделениям
      4. не требуют распределения


   12.10 К расходам предприятия относятся:
      1. стоимость закупленных товаров
      2. расходы, распределяемые на себестоимость товаров
      3. расходы будущих периодов
      4. расходы, распределяемые на финансовый результат
      5. Верны утверждения 1,2 и 3
      6. Верны все утверждения


   12.11 Выберите НЕверное утверждение:
      1. Статья расходов – произвольный классификатор, позволяющий сгруппировать схо- ие затраты, одинаково учитываемые в расходах предприятия
      2. Классификатор статей расходов создается индивидуально на каждом предприятии и пределяет правила учета расходов и анализа финансового результата на данном редприятии
      3. Перечень статей расходов обычно утверждается руководством предприятия
      4. Классификация расходов по статьям является обязательной для использования


   12.12 Себестоимость проданных товаров фиксируется в системе:
      1. вручную
      2. рассчитывается автоматически
      3. при проведении документа "Расчет себестоимости"


   12.13 Документ "Расчет себестоимости"…
      1. формируется автоматически при помощи регламентного задания "Расчет себестои- ости"
      2. формируется автоматически при выполнении обработки "Закрытие месяца"
      3. формируется и проводится менеджером вручную
      4. Такого документа в программе нет, данные о рассчитанной себестоимости можно айти в отчете "Себестоимость товаров"
      5. Возможен либо первый, либо второй вариант (в зависимости от вида расчета себе- тоимости)


   12.14 Сколько документов по расчету себестоимости может быть зарегист- ировано в системе в течение месяца по каждой организации?
      1. только один документ
      2. два документа: предварительный и фактический расчет
      3. сколь угодно много


   12.15 Для получения предварительной оценки себестоимости нужно:

      2. настроить расписание регламентного задания "Расчет себестоимости" в настройках дминистрирования
      3. установить функциональную опцию "Рассчитывать предварительную себестои- ость регламентным заданием"
      4. настроить расписание регламентного задания по расчету предварительной себе- тоимости
      5. Последовательно выполнить действия, указанные в пунктах 1 и 2
      6. Последовательно выполнить действия, указанные в пунктах 3 и 4


   12.16 Какие методы оценки стоимости товаров реализованы в системе?

      2. ФИФО (скользящая оценка)
      3. Средняя за месяц
      4. Верны утверждения 1 и 2
      5. Верны утверждения 1 и 3
      6. Верны все утверждения


   12.17 Как рассчитывается предварительная оценка себестоимости?
      1. По методу ФИФО (средневзвешенная оценка)
      2. По методу ФИФО (скользящая оценка)
      3. По методу "Средняя за месяц"
      4. В настройке предварительного расчета себестоимости можно указать любой из пе- ечисленных выше вариантов для каждой организации


   12.18 По какому методу осуществляется фактический расчет себестоимо- ти?
      1. По методу ФИФО (средневзвешенная оценка)
      2. По методу ФИФО (скользящая оценка)
      3. По методу "Средняя за месяц"
      4. В настройке фактического расчета себестоимости можно указать любой из перечис- енных выше вариантов для каждой организации


   12.19 Фактический расчет себестоимости выполняется…
      1. настройкой расписания и выполнения регламентного задания "Расчет себестоимо- ти"
      2. регламентным заданием по расчету фактической себестоимости, с установленной ункциональной опцией "Рассчитывать фактическую себестоимость регламентным аданием"
      3. проведением документа "Расчет себестоимости " с установленной опцией "Факти- еский"
      4. в рамках рабочего места "Закрытие месяца", которое выполняет все действия по за- рытию отчетного периода


   12.20 Процедуры в рамках рабочего места "Закрытие месяца"…
      1. могут исполняться в произвольном порядке
      2. должны исполняться последовательно снизу вверх
      3. должны исполняться последовательно сверху вниз
      4. должны исполняться снизу вверх или сверху вниз – зависит от выбора функцио- альных настроек


   12.21 Для анализа себестоимости проданных товаров удобно использовать тчет

   12.22 Механизм партионного учета…
      1. в программе не предусмотрен
      2. позволяет вести учет себестоимости товаров в разрезе документов поступления
      (формирующих учетную партию) и рассчитывать себестоимость по методу ФИФО
      (скользящая оценка)
      3. позволяет вести учет себестоимости товаров в разрезе документов поступления и ассчитывать себестоимость по методу ФИФО (взвешенная оценка)
      4. позволяет вести учет себестоимости товаров в разрезе документов поступления и ассчитывать себестоимость по методу "Средняя за месяц"


   12.23 Отказ от ведения партионного учета…
      1. не возможен – пользователь устанавливает соответствующую функциональную оп- ию только один раз
      2. возможен только при отсутствии документа "Расчет себестоимости" по методу ИФО (скользящая оценка)
      3. возможен в любом случае
      4. возможен, если у пользователя есть соответствующие права
      5. не возможен, т.к. в программе отсутствуют партионный учет
      6. возможен только при отсутствии документа "Расчет себестоимости" по методу ИФО (взвешенная оценка)
