   10.1 Взаиморасчеты с партнерами ведутся со следующей детализацией:
      1. Организация, контрагент
      2. Организация, контрагент, партнер
      3. Организация, контрагент, партнер, договор
      4. Организация, контрагент, партнер, заказ
      5. Организация, контрагент, партнер, накладная
      6. Возможны варианты 3, 4, 5 в зависимости от порядка расчетов  (TRUE_FLAG)


   10.2 В качестве объекта расчетов могут выступать:
      1. Договор, заказ, первичный документ (накладная)  (TRUE_FLAG)
      2. Индивидуальное соглашение, договор, заказ
      3. Индивидуальное соглашение, договор, заказ, первичный документ (накладная)
      4. Заказ, первичный документ (накладная)
      5. Организация, контрагент, партнер, накладная.


   10.3 Можно ли отнести аванс на договор партнера и произвести зачет аван- а по документу расчета?
      1. Да, для этого нужно создать договор с выбранным порядком расчетов по заказам ли по накладным, затем отнести денежные средства на заказ или накладную, когда ни будут созданы  (TRUE_FLAG)
      2. Да, если у пользователя есть соответствующие права
      3. Нет, при ведении расчетов по договорам фиксируется только общая задолженность о договору


   10.4 Изначально в индивидуальном соглашении с клиентом был установлен ариант расчетов "по договору". Но вдруг возникла необходимость заре- истрировать оплату по конкретной накладной. Можно ли изменить по- ядок расчетов?
      1. Да, в карточке документа, по которому нужно зарегистрировать оплату в отдельном иалоговом окне "Правила оплаты"  (TRUE_FLAG)
      2. Да, на закладке "Дополнительно" документа, по которому нужно зарегистрировать плату
      3. Нет, вариант расчета устанавливается либо в соглашении, либо в договоре
      4. Возможны варианты 1 и 2


   10.5 Порядок расчетов…
      1. изменяется только в сторону увеличения детализации расчетов: если был установ- ен порядок расчетов "по договору", то он может быть изменен на порядок расчетов "по заказам" или "по накладным"  (TRUE_FLAG)
      2. изменяется в сторону уменьшения детализации расчетов: если был установлен по- ядок расчетов "по заказам", то его можно изменить на порядок расчетов "по догово- ам".
      3. изменяется либо в сторону увеличения детализации расчетов, либо в сторону меньшения детализации
      4. определяется либо в соглашении, либо в договоре и не меняется в документах


   10.6 Какая из перечисленных аналитик будет являться обязательной при едении взаиморасчетов?
      1. Партнер  (TRUE_FLAG)
      2. Соглашение с клиентом
      3. Договор с клиентом
      4. Заказ клиента
      5. Накладная


   10.7 Изначально был установлен вариант расчетов "по заказам". Можно ли осмотреть оплату по конкретной накладной?
      1. Нет, можно увидеть только информацию по заказу клиента
      2. Нет, можно увидеть информацию только по договору с клиентом
      3. Да, если накладная оформлена без заказа  (TRUE_FLAG)
      4. Можно в любом случае


   10.8 В договоре с покупателем изначально выбрана детализация расчетов "по договорам". Можно ли посмотреть оплату по конкретной накладной, формленной в рамках этого договора?
      1. Нет, можно увидеть только информацию по заказу клиента
      2. Нет, можно увидеть информацию только по договору с клиентом  (TRUE_FLAG)
      3. Да, если накладная оформлена без заказа
      4. Можно в любом случае


   10.9 Можно ли запретить отгрузку клиенту при превышении суммы или срока задолженности?
      1. Да, эта информация указывается в соглашении
      2. Да, эта информация указывается в договоре с клиентом  (TRUE_FLAG)
      3. Да, если у пользователя есть соответствующие права
      4. Нет, такой возможности в программе не предусмотрено


   10.10 Можно ли перенести аванс с одного заказа клиента на другой, если они формлены по разным договорам?
      1. Да, при помощи кнопки "Зачет оплаты"
      2. Да, при помощи документа "Взаимозачет задолженности"  (TRUE_FLAG)
      3. Да, если есть соответствующие права
      4. Нет, такая возможность не предусмотрена
      5. Да, при помощи документа "Корректировка долга"


   10.11 Выберите НЕверное утверждение:
      1. Форма "Зачет оплат по документу" используется для различных вариантов зачета ванса
      2. Форма "Зачет оплат по документу" вызывается из формы соответствующих доку- ентов по команде "Зачет оплаты"
      3. На закладке "Нераспределенные платежи" формы "Зачет оплат по документу" отображаются в числе прочих документы с пустым объектом расчетов вне зависимости от даты
      4. На закладке "Нераспределенные платежи" формы "Зачет оплат по документу" отображаются в числе прочих документы с датой большей, чем дата зачитываемого документа
      5. На закладке "Неиспользованные авансы" формы "Зачет оплат по документу" отображаются документы с датой, предшествующей дате зачитываемого документа  (TRUE_FLAG)


   10.12 Выберите ВЕРНОЕ утверждение:
      1. Валюта документа продажи определяется строго по валюте договора
      2. Документы продажи можно оформлять в валюте взаиморасчетов и валюте регла- ентированного учета
      3. Документы продажи можно оформлять в валюте взаиморасчетов и валюте управ- енческого учета
      4. Документы продажи можно оформлять в любой валюте, независимо от валюты заиморасчетов  (TRUE_FLAG)


   10.13 В каком отчете можно проанализировать просроченную дебиторскую адолженность?

      4. Верны утверждения 1 и 3

   10.14 Задолженность по взаиморасчетам, которая никогда не будет погаше- а, оформляется документом__

   10.15 Документ "Сверка взаиморасчетов может быть сформирован:
      1. по взаиморасчетам конкретной организации и конкретного контрагента  (TRUE_FLAG)
      2. по взаиморасчетам всех наших организаций и конкретного контрагента
      3. по взаиморасчетам конкретной организации и всех контрагентов конкретного партнера
      4. по взаиморасчетам всех наших организаций и всех контрагентов конкретного партнера

   10.16 В документе "Сверка взаиморасчетов"….
      1. после согласования сверки взаиморасчетов формируется печатная форма "Акт сверки взаиморасчетов"
      2. предусмотрена возможность группового создания актов сверки с использованием пециального помощника
      3. есть возможность проанализировать отклонения в результате двустороннего обмена анными с партнером
      4. Верны утверждения 1 и 2
      5. Верны все утверждения  (TRUE_FLAG)


   10.17 Данные о подотчетном лице регистрируются…

      3. В справочнике "Контактные лица"
      4. В справочнике "Партнеры"


   10.18 Подотчетнику выдаются …

      2. только безналичные денежные средства
      3. наличные денежные средства и безналичные  (TRUE_FLAG)


   10.19 Товары, закупленные подотчетником...
      1. нельзя оприходовать на складе до составления авансового отчета
      2. оприходуются на складе, но доступны для продажи только после составления аван- ового отчета
      3. оприходуются на складе документом "Приходный ордер на товары" с установлен- ым видом операции "От подотчетника"
      4. можно продать, если оформлен документ "Поступление товаров и услуг" с установ- енным видом операции "Закупка через подотчетное лицо" и документ "Приходный рдер на товары" в случае использования на складе ордерной схемы  (TRUE_FLAG)
      5. можно продать, если оформлен документ "Поступление товаров и услуг" с установ- енным видом операции "Закупка через подотчетное лицо" и документ "Приходный рдер на товары" с установленным видом операции "От подотчетника" в случае ис- ользования на складе ордерной схемы


   10.20 Если подотчетное лицо произвело закупку товаров у разных постав- иков, то в этом случае…
      1. составляется отдельный авансовый отчет по каждому документу поступления
      2. можно составить один авансовый отчет, но для этого нужно установить соответст- ующую функциональную опцию в разделе "НСИ и Администрирование"
      3. составляется отдельный авансовый отчет по каждому поставщику
      4. составляется один авансовый отчет, т.к. в документе указываются не товары, а данные о расходах  (TRUE_FLAG)


   10.21 Выберите ВЕРНОЕ утверждение:

      2. Валюта взаиморасчетов с подотчетным лицом определяется в документе "РКО" при ыдаче денежных средств подотчетному лицу  (TRUE_FLAG)
      3. Валюта взаиморасчетов с подотчетным лицом определяется в документе "Авансовый отчет"
      4. В качестве валюты взаиморасчетов с подотчетным лицом используется валюта правленческого учета
      5. В качестве валюты взаиморасчетов с подотчетным лицом используется валюта регламентированного учета


   10.22 Авансовый отчет …
      1. может быть создан в статусах "Подготовлен" и "Утвержден"
      2. может быть создан в статусах "Не согласован", "Согласован", "Утвержден"
      3. может быть создан в статусах "Подготовлен" и "Утвержден", если установлена соответствующая функциональная опция, в противном случае документ создается без статусов  (TRUE_FLAG)
      4. может быть создан в статусах "Не согласован", "Согласован", "Утвержден", если установлена соответствующая функциональная опция, в противном случае документ создается без статусов
      5. создается без статусов – статусов для документа "Авансовый отчет" не предусмотрено


   10.23 В документе "Авансовый отчет" можно…
      1. заполнить информацию об использованных/полученных авансах и прочих расходах а закладке "Расходы" по команде "Заполнить полученными авансами"
      2. заполнить информацию об оплатах поставщику на закладке "Оплата поставщикам"
      3. заполнить на закладке "Товары" информацию о товарах, закупка которых была совершена подотчетным лицом
      4. Верны утверждения 1 и 2  (TRUE_FLAG)
      5. Верны все утверждения


   10.24 Выберите НЕверное утверждение:
      1. Для оформления закупки подотчетным лицом товаров предусмотрен документ "По- тупление товаров и услуг" с типом операции "Закупка через подотчетное лицо"
      2. Для оформления операции по покупке прочих услуг, нематериальных и внеоборот- ых активов подотчетным лицом предусмотрен документ "Поступление услуг и про- их активов" с типом операции "Закупка через подотчетное лицо"
      3. Документы "Поступление товаров и услуг" и "Поступление услуг и прочих активов"  типом операции "Закупка через подотчетное лицо" являются источниками данных для ввода документа "Авансовый отчет", но не являются источником данных для формирования печатной формы "Авансовый отчет"
      4. Документы "Поступление товаров и услуг" и "Поступление услуг и прочих активов"  типом операции "Закупка через подотчетное лицо" являются источниками данных для формирования отчета "Контроль операций с денежными средствами"
      5. Документы "Поступление товаров и услуг" и "Поступление услуг и прочих активов"  типом операции "Закупка через подотчетное лицо" не являются источниками данных для ввода документа "Авансовый отчет" и для формирования печатной формы "Авансовый отчет"  (TRUE_FLAG)


   10.25 При помощи рабочего места "Авансовые отчеты" можно …

      2. создать документ "Поступление услуг и прочих активов" с типом операции "Закупка через подотчетное лицо"

      4. оформить необходимую операцию на основании авансового отчета при помощи со- тветствующей кнопки
      5. Верны утверждения 1, 2, 4
      6. Верны все утверждения.  (TRUE_FLAG)

      2. отображаются на закладке "К поступлению" в списке документов "Приходные кас- овые ордера" и на закладке "К поступлению" в журнале "Безналичные платежи"  (TRUE_FLAG)
      3. отображаются на закладке "Распоряжения на поступления" в списке документов "Расходные кассовые ордера" и на закладке "К поступлению" в журнале "Безналичные латежи"
      4. Верны утверждения 1 и 2
      5. Верны утверждения 1 и 3


   10.28 Поле для указания срока предоставления авансового отчета есть в окументах…

   10.29 Можно ли контролировать денежные средства у подотчетника в раз- езе статей движения денежных средств?
      1. Нет, такой возможности в программе пока не предусмотрено

   10.30 Возврат прерасходованных денежных средств подотчетнику_

   10.31 Возврат неизрасходованных денежных средств подотчетником формляется:

      2. документом "Поступление безналичных ДС" с видом операции "Возврат от подотчетника"
      3. Возможны оба варианта. Зависит от того, каким образом выдавались деньги подотчетнику  (TRUE_FLAG)
      4. Возможны оба варианта, независимо от того, каким образом выдавались деньги одотчетнику


   10.32 В случае выдачи перерасхода денежных средств подотчетнику…
      1. нужно в любом случае предварительно оформить заявку на расходование денежных редств с операцией "Выдача подотчетнику"
      2. нужно предварительно оформить заявку на расходование денежных средств с операцией "Выдача подотчетнику", если для банковского счета (кассы) установлена оп- ия использования заявок на расходование денежных средств  (TRUE_FLAG)
      3. можно обойтись без заявки на расходование денежных средств, если у пользователя есть соответствующие права на выдачу денежных средств
      4. можно обойтись без заявки на расходование денежных средств в любом случае


   10.33 Товары, закупленные подотчетником, до оформления им авансового тчета…
      1. нельзя вернуть поставщику
      2. можно оприходовать на складе, но нельзя вернуть их поставщику
      3. можно вернуть поставщику, оформив документ "Расходный ордер на товары" с ус- ановленным видом операции "От подотчетника"
      4. можно вернуть поставщику, оформив документ "Возврат товаров поставщику" и "Расходный ордер на товары" в случае использования ордерной схемы на складе  (TRUE_FLAG)

      4. Можно, для этого перерасход денежных средств необходимо зафиксировать в документе "Авансовый отчет" на закладке "Использованные авансы"
