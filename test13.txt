   13.1 Выберите ВЕРНОЕ утверждение:
      1. Пользователь имеет право вносить любые изменения в типовую конфигурацию, а акже разрабатывать собственные конфигурации, используя типовую конфигурацию в ачестве образца  (TRUE_FLAG)
      2. Пользователь имеет право только использовать типовую конфигурацию, не внося в ее изменений, поскольку она защищена авторским правом разработчиков
      3. Пользователь имеет право изменять в типовой конфигурации печатные формы, под- лючать новые печатные формы и внешние отчеты и обработки
      4. Пользователь имеет право использовать ее только как образец и пример для разра- отки собственных конфигураций, сохраняя первоначальную настройку в качестве талона


   13.2 Резервное копирование информационной базы…
      1. не предусмотрено, администратор системы может в конфигураторе выгрузить кон- игурацию и информационную базу в файл для сохранности данных
      2. настраивается в режиме "1С:Предприятие" и осуществляется при завершении рабо- ы администратора
      3. настраивается в режиме "1С:Предприятие"осуществляется по расписанию, создан- ому администратором
      4. Верны варианты 2 и 3, все зависит от настройки резервного копирования  (TRUE_FLAG)


   13.3 К каким последствиям приведет преждевременное прекращение про- есса выгрузки информационной базы в архивный файл?
      1. Процесс выгрузки нельзя прерывать преждевременно, поскольку это приведет к астичному или полному разрушению архивируемой базы данных
      2. Прерывание процесса выгрузки не приведет к разрушению архивируемой базы дан- ых, однако созданный архив нельзя будет использовать для восстановления данных  (TRUE_FLAG)
      3. Процесс выгрузки можно прервать преждевременно, это не приведет к разрушению рхивируемой базы данных, впоследствии этот процесс можно продолжить, например,  следующем сеансе работы с программой, начиная с точки прерывания
      4. Процесс выгрузки можно прервать преждевременно, это не приведет к разрушению рхивируемой базы данных, а созданный (неполный) архив можно использовать для астичного восстановления данных


   13.4 После выполнения операции восстановления данных из архивного айла…
      1. информация текущей базы данных будет полностью заменена информацией из ар- ивной копии и не может быть восстановлена  (TRUE_FLAG)
      2. информация текущей базы данных будет полностью заменена информацией из ар- ивной копии, но она может быть восстановлена при помощи операции "отката" к редыдущему варианту
      3. информация из архивной копии может быть восстановлена только в пустой дирек- ории, поэтому текущую базу данных предварительно потребуется удалить или соз- ать новый каталог
      4. информация из архивной копии может быть восстановлена только в пустом катало- е, поэтому программа автоматически создаст новый каталог для восстановленной ба- ы данных
