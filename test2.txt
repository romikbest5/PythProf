   2.1 Ценовые группы используются:
      1. для группировки товаров в прайс-листе
      2. для назначения различных видов цен по ценовым группам при отгрузке товаров по- упателям
      3. для назначения скидок и наценок для товаров одной ценовой группы
      4. для назначения одинаковых цен продажи товарам одной ценовой группы
      5. Возможно применение всех вариантов
      6. Возможно применение вариантов 1, 2 и 3  (TRUE_FLAG)


   2.2 Выберите ВЕРНОЕ утверждение:
      1. Одна номенклатурная позиция может быть отнесена только к одной ценовой группе  (TRUE_FLAG)
      2. Одна номенклатурная позиция может входить в несколько ценовых групп
      3. Одна номенклатурная позиция может быть отнесена только к одной ценовой груп- е, а может входить в несколько ценовых групп. Это зависит от функциональной оп- ии "Использование нескольких ценовых групп для одной номенклатуры" ("НСИ и дминистрирование – Настройка разделов")


   2.3 Сколько видов цен может задать пользователь?
      1. Пользователь может задать только один вид цены
      2. Пользователь всегда может задать несколько видов цен
      3. Пользователь всегда может задать несколько видов цен, но не более 5-ти
      4. В зависимости от установки функциональной опции "Несколько видов цен" пользо- атель может использовать либо один вид цены, либо несколько  (TRUE_FLAG)


   2.4 Установка флажка "При вводе на основании документов поставки" в арточке вида цены означает, что...
      1. при вводе документов поставки система автоматически установит такие цены на то- ары
      2. при вводе документа "Установка цен номенклатуры" на основании документов по- тавки такие виды цен будут выбраны автоматически (отмечены флажками)  (TRUE_FLAG)
      3. после проведения документов поставки система автоматически предложит устано- ить такие цены на товары
      4. после проведения документа поставки система автоматически откроет форму доку- ента "Установка цен номенклатуры" с уже рассчитанными ценами с данными при- наком


     2.5 Если в карточке вида цены не установлены признаки "При продаже кли- нтам" и "При передаче между организациями", то...
        1. такой вид цен автоматически не подставляется в документы продажи/передачи то- аров между организациями, но может быть выбран вручную
        2. такой вид цен нельзя применить при продаже/передаче товаров между организации, о можно использовать в качестве вспомогательного при расчете, например, цены родажи  (TRUE_FLAG)
        3. такой вид цен считается устаревшим и недействующим


     2.6 Что в обязательном порядке включает в себя любой вид цены?
        1. валюту и способ задания цены
        2. валюту и признак включения НДС в цену  (TRUE_FLAG)
        3. валюту и правила округления цен
        4. способ задания цены и признак включения НДС в цену


     2.7 Наценки для товаров для расчета отпускных цен регистрируются:

        3. Вид цены, у которого обязательно должны быть установлены признаки "При про- аже клиентам" и "Цена включает НДС"
        4. Вид цены с установленным признаком "При продаже клиентам" и валютой – рубли


   2.10 Использование упрощенных возможностей позволяет:
      1. вводить правила расчета в упрощенном режиме: без детализации наценок по цено- ым группам, использования произвольных правил расчета, механизма психологиче- кого округления и др.  (TRUE_FLAG)
      2. вводить правила расчета в упрощенном режиме: без использования таких правил асчета как наценка на цену поступления и на другой вид цен
      3. вводить правила расчета в упрощенном режиме: рассчитывать цены только вруч- ую, без детализации наценок по ценовым группам, механизма округления
      4. вводить правила расчета в упрощенном режиме: без детализации наценок по цено- ым группам, механизма округления

      3. Да, но только для видов цен со способом задания цены "Ручное назначение", "На- енка на другой вид цен" или "Произвольная формула расчета"  (TRUE_FLAG)
      4. Да, но только для видов цен со способом задания цены "Ручное назначение", "На- енка на цену поступления" или "Произвольный запрос к ИБ"

     2.16 Для товаров ценовой группы "Вентиляторы" установлен порог сраба- ывания 3%. Новые цены на товары этой группы превышают старые на 1,5%. В этом случае:

        2. Система выдаст предупреждение, что новые цены отличаются от старых менее чем а 3% и спросит, нужно ли регистрировать новые цены или нет
        3. Система не будет регистрировать новые цены, оставит старые  (TRUE_FLAG)

        4. Можно назначить столько значений оптовой цены, сколько различных упаковок за- ано для товара
        5. Справедливы утверждения, указанные в пунктах 3 и 4

        4. на основании соглашения с клиентом

   2.20 Для партнера не определены цены, по которым ему отгружается товар. акой вид цен будет установлен по умолчанию при выборе партнера в окументе "Реализация товаров и услуг"?

      3. Значения вида цены в документе заполнено не будет  (TRUE_FLAG)

      2. Формируется документ "Установка цен номенклатуры" по заранее установленным араметрам отбора номенклатуры и списку цен, которые нужно задать

      4. можно создать из рабочего места "Цены(Прайс-лист)", либо вручную из списка до- ументов "Установка цен номенклатуры", либо из документа поступления, либо из окумента "Регистрация цен поставщиков"


   2.23 Менеджер при помощи рабочего места "Цены(прайс-лист)" изменяет ены для продажи на товары группы "Бытовая техника" на 3%. При этом становлен флажок "Рассчитывать цены автоматически". Что произой- ет в этом случае?
      1. Будут автоматически пересчитаны все цены на товары из этой группы, независимо т того, какие виды цен выбрал менеджер
      2. Будут автоматически пересчитаны только те цены, которые выбрал менеджер
      3. В соответствии с изменениями выбранных цен, будут пересчитаны и зависимые от их  (TRUE_FLAG)
